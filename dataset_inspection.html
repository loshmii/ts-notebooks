Distribution of the categories to be classified

We see that the classes we want to classify are highly imbalanced. If we were to follow the rule that any class whose share in the dataset is less than 5% should be removed, we would end up with only two classes we could diagnose (Blood Donor and Cirrhosis), which is unacceptable. Since these classes represent different stages of liver pathology, we decided to merge all classes representing pathological conditions into a single class, while removing the Suspect Blood Donor class because it cannot be reliably assigned to either of the two newly formed classes, and its number of instances is too small to significantly affect the classification results.

sex_series = df['Sex'].astype(str).str.strip().str.lower()

category_map = {
    "0=Blood Donor": "blood_donor",
    "0s=suspect Blood Donor": "suspect_blood_donor",
    "1=Hepatitis": "hepatitis",
    "2=Fibrosis": "fibrosis",
    "3=Cirrhosis": "cirrhosis",
}
df['category_norm'] = df['Category'].map(category_map).fillna('unknown').astype('category')
df = df[df['category_norm'] != 'suspect_blood_donor'].copy()
df['class'] = df['category_norm'].isin({'hepatitis','fibrosis','cirrhosis'}).astype(np.int8)

Mapping the categorical random variables

We map gender to 0/1, where 1 indicates a male sample and 0 otherwise. This encoding is chosen because the models that use such features transform categorical representations into vector representations. For example, the Tabular Classifier class, which is essentially a neural network, maps categorical classes into 3-dimensional vector embeddings that are learned during training. In contrast, models such as KNN and Tree classifiers convert these categories into distinct one-hot vectors to avoid introducing bias during classification.
