Classes

In order to properly create KNN classifier, we have created KNNSpec and KNNConfig helper classes, which represent dataset and architecture hyperparameter specification. Using these two classes, the function evaluate_knn_cnn performs evaluation of different classificator configurations over a specified grid, performs comparison over the validation set, and picks the best configuration based on the specified metric which is then evaluated on the test set. Likewise, using Encoders class we perform data normalization using training set metrics. Specifically:

$$z_i = \dfrac{x_i - \mu_{\text{train}}}{\sigma_{\text{train}}}$$

$$\mu_{\text{train}} = \dfrac{1}{N_{\text{train}}} \sum_{j=1}^{N_{\text{train}}} x_j$$

$$\sigma_{\text{train}} = \sqrt{ \dfrac{1}{N_{\text{train}}} \sum_{j=1}^{N_{\text{train}}} (x_j - \mu_{\text{train}})^2 }$$

$$z = \dfrac{x - \mu_{\text{train}}}{\sigma_{\text{train}}}$$

Classifier Evaluation

The classifier was evaluated using k-fold cross-validation. The input dataset was divided into k subsets. We train the classifier on k1 subsets and evaluate it on the remaining one.

This procedure is repeated for every possible choice of the held-out subset. The overall complexity is O(k ), where O() is the cost of fitting and evaluating the classifier for a single subset.

Result Analysis

We observe that the accuracy decreases as the number of neighbors increases beyond the optimal value of 3. This behavior is expected given the distribution of the input dataset. Specifically, when the ratio of labels is approximately 7:1, the KNN classifier is not an ideal classification method because the labels are unevenly distributed in the feature space.
